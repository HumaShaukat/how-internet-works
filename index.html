🔖
Ephemeral Port
Temporary client port. Randomly chosen by the device's OS. Born at session start, discarded when done. Range: 1024–65535.
Ports
🚪
Well-Known Port
Fixed server port. Same forever. Port 443 = HTTPS. Port 53 = DNS. Port 80 = HTTP. These never change — they are industry standard.
Ports
🔌
Socket
The complete address of one conversation: Source IP + Source Port + Destination IP + Destination Port. Every request on the internet is a socket.

Chapter 04
The Full Journey

You type google.com. Here is every single thing that happens before the page loads — step by step.

From WiFi Toggle → Google Homepage
Click "Next Step" to walk through the complete journey.
0
WiFi Toggled ON → NIC Activates
You flip WiFi on. The Network Interface Card (NIC) wakes up. Your device has no IP address yet — it cannot communicate with anyone.
// Device state: NO IP ADDRESS // NIC status: ACTIVE // Next: Start DORA conversation
1
DORA — Device Gets Its Configuration
Your device broadcasts a Discover. DHCP server offers an IP. Device requests it. Server acknowledges. Your device now has everything it needs.
// DHCP Acknowledge received: // Private IP: 192.168.1.3 // Subnet Mask: 255.255.255.0 // Default Gateway: 192.168.1.1 ← router's private IP // DNS Server: 8.8.8.8 ← DHCP introduced DNS
2
You Type google.com
Your browser receives the name "google.com". But the internet only understands IP numbers. Your device must translate this name first — before anything else can happen.
// Browser: "I need to connect to google.com" // Device: "I don't know that IP. I must ask DNS." // Device opens ephemeral port: 5001 (bookmark for this session) // DNS server address: 8.8.8.8 (told by DHCP)
3
DNS Request — Conversation 1
Device sends a DNS request. It goes to the router first (Default Gateway). Router runs PAT — swaps private IP for public IP. Packet travels the internet hop by hop to 8.8.8.8.
// Packet leaving device: // FROM: 192.168.1.3 : 5001 → TO: 8.8.8.8 : 53 // // Router PAT translation: // FROM: 203.45.67.89 : 5001 → TO: 8.8.8.8 : 53 // NAT Table: port 5001 = 192.168.1.3 (laptop)
4
DNS Reply — Name Translated
8.8.8.8 replies with Google's IP address. Reply travels back to your router. Router checks NAT Table — port 5001 belongs to your laptop. Delivers the reply. Port 5001 closes.
// DNS reply: "google.com = 142.250.70.46" // Reply addressed to: 203.45.67.89 : 5001 // Router NAT Table: port 5001 → 192.168.1.3 (laptop) // Delivered. Port 5001 CLOSED. Session over.
5
HTTPS Request — Conversation 2
Device now knows Google's IP. Opens a new ephemeral port (5002 — new bookmark, new session). Sends HTTPS request to Google's web server on Port 443. Router runs PAT again.
// Packet leaving device: // FROM: 192.168.1.3 : 5002 → TO: 142.250.70.46 : 443 // // Router PAT translation: // FROM: 203.45.67.89 : 5002 → TO: 142.250.70.46 : 443 // NAT Table: port 5002 = 192.168.1.3 (laptop)
6
Google Replies — Page Loads ✅
Google's web server sends back the homepage. Reply travels back to your router. Router checks NAT Table — port 5002 = your laptop. Delivers the webpage. Port 5002 closes. You see Google.
// Google reply: homepage HTML // Reply to: 203.45.67.89 : 5002 // Router: port 5002 → 192.168.1.3 (laptop) // Page delivered. Port 5002 CLOSED. // // Total ephemeral ports used: 2 (5001 for DNS, 5002 for HTTPS) // Both born and died for this one page load. ✅

Chapter 05
Key Concepts

Every term from this explainer — defined simply.

🌐
IP Address
A device's address on a network. Temporary — given by DHCP. Without one, a device cannot send or receive anything.
Networking
🔄
DHCP
Dynamic Host Configuration Protocol. Automatically gives devices their IP address, subnet mask, default gateway, and DNS server address.
Protocol
🏠
LAN
Local Area Network. A network inside one building. The internet is millions of LANs connected through routers.
Networking
🔀
NAT / PAT
Network/Port Address Translation. Software inside the router that translates Private IP + Port → Public IP + Port for every outgoing request.
Router
📖
NAT Table
The router's diary. Records which device made which request using port numbers. Used to deliver replies to the correct device.
Router
🔍
DNS
Domain Name System. Translates human-readable names (google.com) into IP numbers. Always uses Port 53. Introduced to your device by DHCP.
Protocol
🚨
Rogue DHCP
A hacker's device pretending to be a DHCP server. If it replies faster than the real server, it can inject malicious DNS and gateway config — a Man in the Middle attack.
Security
🛡️
DHCP Snooping
A switch security feature that only allows trusted ports to send DHCP Offers. Blocks rogue DHCP servers before they reach devices.
Security
🔌
Socket
The complete address of one internet conversation: Source IP + Source Port + Destination IP + Destination Port. Every request is a socket.
